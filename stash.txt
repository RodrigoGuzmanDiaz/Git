ahora vamos a ver stash
el stash es usualmente utilizado para el cuando necesito hacer salir del lugar en donde me encuentro para continuar trabajando en otra rama el punto es 

porgamos un ejemplo por ejemplo estamos trabajando en una funiconalidad y en dado caso un compañero de trabajo nos pide ayuda ya sea porque se cayo un servicio, hay una invulnerabilidad que necesita ser arreglada, etc. el punto es que tenemos cambios que estan a media por lo tanto no es viable hacer commit porque recordemos que solo debemos hacer cambios cuando tengas cambios importante para la aplicacion o avances significativos. 
para esto existe stash 
stash es un area entonces se guarda los cambios no han sido guardados en commit 

recordemos que no se nos permite cambiar de rama si cambios sin guardar en este caso usamos stash 

por ejemplo 
Trabajando en la rama "funcionalidad-pago"
en esta rama tengo cambios a medias que no ameritan hacerles commit por lo tanto lo ideal es usar stash
para hacer stash se usa el siguiente comando:
git stash

ahora 
marelin@marelin-HP-240-G7-Notebook-PC:~/Documentos/curso-git/Actividades-curso-git/07-demo-stash$ git stash
Directorio de trabajo y estado de índice WIP on funcionalidad-pago: 5440fe5 Resolviendo conflictos guardados

ahora vemos que los cambios que teniamos ya no estan esto es porque se guardaron en el area de stash

ahora si podemos cambiar de rama para prestar nuesra atencion en otra cosa
git switch otra-rama

"despues de trabajar en el otro lugar ya que los errores se hayan resuelto o que se haya solucionado ya podemos regresar a nuestra rama y continuar trabajando"

git switch funcionalidad-pago

en este punto ya podemos continuar trabajando en lo que estabamos haciendo antes

¿Pero como hago para tener mis cambios anteriores ?
el comando para hacer esto es:
git stash pop

---pro tip---
en dado caso es posible que se presente el caso de que se nos olvida que tenemos algo en stash es posible verificarlo antes que cometamos un error y es muy recomendable que usemos esto si no estamos seguron si hay algo en stash o no
usando el siguiente comando se pueden ver las referencia a stash que tenemos
git stash list
stash@{0}: WIP on funcionalidad-pago: 5440fe5 Resolviendo conflictos

en el caso de usar stach tambien podemos almacenar multiples stash PERO LO RECOMENDABLE ES USAR SOLAMENTE UN STASH ya que si trabajamos con mas de uno esto puede generar conflictos

----Advertencias----
esto solo funciona con ARCHIVOS MODIFICADOS esto NO FUNCIONA CON ARCHIVO NUEVOS 
ES IMPORTANTE TOMAR ESO EN CUENTA

---Alternativa a git stash----
Comandos clave
Guardar todo (incluyendo archivos nuevos):
git stash -u

# O también:
git stash --include-untracked

Guardar archivos nuevos + archivos ignorados:
git stash -a

O también:
git stash --all

Recuperar cambios:
git stash pop
