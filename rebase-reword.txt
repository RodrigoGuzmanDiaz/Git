para el caso del git rebase con la variante de reword sirve exclusivamente para el cambio de mensaje de commit, es imrpotante decir es que mejor no hacer el mensaje de commit porque al hacerlo en repositorios colaborativos esto puede generar conflictos, por lo tanto es recomendable usarlo cuando nuestros cambios aun no estan publicados

primero usamos el comando de git rebase -i HEAD~<numero_de_commits>

pro ejemplo si queremos hacerlo con 3 commit seria algo asi*/
git rebase -i HEAD~3
