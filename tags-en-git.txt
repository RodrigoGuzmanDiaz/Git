¿Qué es un tag en Git?

Un tag en Git es una forma de marcar o etiquetar un punto específico en la historia de tu proyecto, usualmente un commit importante. Imagina que tienes una línea de tiempo con varios cambios (commits), y quieres poner una marca para recordar un momento relevante, por ejemplo, cuando lanzaste la versión 1.0.0 de tu aplicación. Ese marcador es un tag.

Los tags son útiles para identificar versiones de tu software, como versiones de lanzamiento, correcciones importantes, o hitos en el desarrollo. No cambian la historia ni el código, solo funcionan como "etiquetas" que apuntan a un commit específico.

Ejemplo sencillo de tags

Supongamos que tu proyecto tiene estos 4 commits en orden cronológico:

Commit 1: Inicio del proyecto

Commit 2: Unión de trabajos realizados por diferentes personas

Commit 3: Nuevas funcionalidades añadidas

Commit 4: Creación del ejecutable (archivo listo para usar)

Para indicar que el commit 4 es la versión 1.0.0 de tu aplicación, creas un tag llamado v1.0.0 que apunta a ese commit. Así, cuando quieres regresar o referirte a esa versión, solo miras el tag.

Comandos básicos para trabajar con tags

Crear un tag ligero:
git tag v1.0.0

Crear un tag en un commit específico:
git tag v1.0.0 <hash_commit>

Ver todos los tags:
git tag

Compartir los tags con el repositorio remoto:
git push origin --tags

Buenas prácticas con tags

Usa nombres claros y consistentes, por ejemplo, siguiendo SemVer (Versionado Semántico) con formatos como v1.0.0, v2.1.3, etc.

Agrega un mensaje al tag para describir la versión (tags anotados). Esto es especialmente útil para documentar qué cambios importantes incluye esa versión:
git tag -a v1.0.0 -m "Versión inicial estable con ejecutable"

Taguea siempre versiones estables o hitos importantes, no cada pequeño cambio.

Sincroniza tus tags con el repositorio remoto para que todo el equipo pueda acceder a ellos.

Algunas compañías usan nombres especiales para tags, como Android que usa nombres de dulces para sus releases (“Pie”, “Oreo”), pero esto es más una convención interna.
